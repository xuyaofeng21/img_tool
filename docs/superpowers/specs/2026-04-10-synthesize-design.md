# 合成标注模块设计

## 概述

在 GUI 中新增 `synthesize` 任务类型，实现智能多边形标注合成功能。该功能将带标注的物体图像合成到背景图中，并自动更新 LabelMe 格式的 JSON 标注文件。

## 功能定位

- **任务类型**: 单次完成的批处理任务（与 bgr2rgb、select_diverse 平级）
- **适用场景**: 训练数据增强——将难以实地采集的物体（如刺猬、电线等）从网上爬取后合成到已有数据集
- **执行模式**: 流式处理（边预处理边合成），复用 TaskManager 的线程池

## 参数设计

### 路径参数

| Key | Label | 说明 | 必填 |
|-----|-------|------|------|
| source_folder | 源物体目录 | 包含待合成物体图片的目录 | 是 |
| bg_json_folder | 背景图目录 | 带 JSON 标注的背景图目录 | 是 |
| output_folder | 输出目录 | 合成结果保存位置 | 是 |

### 业务参数

| Key | Label | 默认值 | 说明 |
|-----|-------|--------|------|
| label | 标注名称 | （必填） | 合成后物体的标注名称 |
| target_label | 源标注标签 | 空 | 从哪个标签抠图（可选） |
| max_objects | 放置数量 | 3 | 每张背景图最多放置物体数 |
| max_object_size | 物体最大尺寸 | 350 | 物体最大边长（像素） |
| model | 抠图模型 | small | small=轻量快 / precise=精准慢 |
| rotation_angle | 最大旋转角度 | 30 | 随机旋转范围（0-指定值） |
| grass_label | 地面标签名 | grass | 表示地面的标签名（可配置） |

## 核心逻辑

### Label 自动决策

```
情况A: 源图片带 JSON（target_label 存在）
  → 直接用 target_label 抠图
  → label 自动等于 target_label（无需用户输入）

情况B: 源图片不带 JSON
  → 使用 rembg 模型抠图
  → label 必须由用户输入
```

### 物体放置规则

1. **地面优先**: 物体放置在 `grass` 标签指定的 polygon 区域内（`grass_label` 参数指定标签名，默认为 "grass"）
2. **障碍避让**: 物体不能完全放置在 `obstacles` 标签的多边形内
3. **随机增强**: 0-度旋转（可配置最大值）+ 水平镜像
4. **防重叠**: 与已放置物体检测碰撞重叠

### 放置区域回退

若背景 JSON 不存在 `grass_label` 标签：
- 回退到图像下半部分（bg_h // 3 以下）
- 日志输出警告信息

## 缓存机制

- **缓存位置**: `.cache/synthesize/`
- **缓存策略**: 基于源文件 mtime 校验有效性
- **过期清理**: 启动时清理 15 天前的缓存
- **手动清除**: 设置页面提供「清除合成缓存」按钮，点击弹窗确认

## 执行流程

```
1. 用户配置参数 → 点击执行
2. 进入流式处理循环:
   for each 背景图:
     → 解析 obstacles_polygons
     → for i in range(max_objects):
         → 随机选择源物体
         → 检查/创建缓存（miss 时调用 rembg 或 polygon 抠图）
         → 检查 grass_label 区域
         → 检查 obstacles 区域（排除完全包含的情况）
         → 检测与已放置物体重叠
         → 放置成功则更新 polygons
     → 保存合成图 + 更新后的 JSON
     → 输出进度: "已处理 X/Y"
3. 任务完成 → 显示成功/失败统计
```

## 进度展示

- 终端日志 tqdm 风格输出：`已处理 5/100`
- 无独立进度条 UI（复用终端日志）

## 错误处理

- 源图片无法读取 → 跳过并记录警告
- 抠图失败 → 跳过该物体，继续处理
- 所有物体放置失败 → 该背景图标记为失败
- JSON 解析失败 → 跳过该背景图
